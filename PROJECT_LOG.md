## PROJECT_LOG.md — рабочий журнал проекта GoNext

Этот файл фиксирует ключевые шаги, решения и изменения относительно плана.

### 1. Общие правила лога

- Не храним здесь секреты, ключи и содержимое `.env`.
- Записи делаем короткими, в формате: `День N, ЧЧ:ММ — что произошло / какое решение принято`.
- Старые записи не переписываем, новые добавляем **в конец**.

### 2. Текущий статус

- MVP по плану из `PLAN.md` (этапы 1–8) реализован: экраны Мест, Поездок, Следующего места, Достопримечательностей и Настроек работают на устройстве и в браузере.
- Локальное хранение данных настроено через SQLite на мобильных платформах (`database.native.ts`), для web используется заглушка (`database.web.ts`), чтобы легче отлаживать интерфейс.
- Основной баг с падением приложения на Android из‑за `SQLite.openDatabase is not a function` устранён, используется новый async‑API `expo-sqlite`.
- Внешний вид главного экрана и форм приведён к аккуратному светлому MVP‑стилю: единый акцентный цвет, закруглённые кнопки, читаемые тексты состояний.
- На всех основных экранах используется общее фоновое изображение через компонент `ScreenBackground`, основной `PrimaryButton` переведён на мягкий градиентный стиль на базе `expo-linear-gradient`, а ключевые элементы интерфейса (поля ввода, переключатели с подсказками, молочные карточки пустых состояний и важного текста) вынесены в отдельные UI‑компоненты, чтобы изменения стиля автоматически применялись ко всем экранам.
