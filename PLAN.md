## PLAN.md — исходный план проекта GoNext

Этот файл фиксирует исходный (стартовый) план проекта.  
В дальнейшем по правилам проекта мы **не переписываем** уже написанное, а только дополняем план ниже по мере развития.

### 1. Текущее состояние

- Проект создан, начальная структура и правила Cursor подключены.
- Исходный функциональный план ещё **не сформулирован** — будет добавлен после постановки первой задачи.

### 2. Черновой каркас плана (заполнится позже)

Ниже будут появляться разделы, когда ты озвучишь первую задачу / промт:

- Общие цели проекта.
- Основные пользовательские сценарии.
- Крупные этапы разработки (MVP, улучшения, интеграции и т.п.).

---

### 2.1. Общие цели проекта (на основе PROJECT.md)

- Реализовать мобильное офлайн‑приложение «Дневник туриста» (GoNext) с локальным хранением данных.
- Дать пользователю удобный способ:
  - вести список интересных мест;
  - планировать поездки с маршрутами;
  - фиксировать реальные события и впечатления в виде дневника.
- Обеспечить простую навигацию по четырём основным режимам:
  - «Места»;
  - «Поездки»;
  - «Следующее место»;
  - «Достопримечательности».

---

### 2.2. Основные пользовательские сценарии (MVP)

1. **Сценарий: управление местами**
   - Добавить новое место (название, описание, координаты, отметки «посетить позже», «понравилось»).
   - Посмотреть список всех мест.
   - Открыть место на карте / в навигаторе.
   - Прикрепить к месту фотографии.

2. **Сценарий: планирование поездки**
   - Создать поездку с названием, описанием и датами.
   - Добавить в поездку места (из списка мест или новые).
   - Упорядочить места в маршруте (изменить порядок).

3. **Сценарий: ведение дневника поездки**
   - Отмечать места в поездке как посещённые.
   - Добавлять к посещённым местам заметки и фото.
   - Переключаться между режимом «план» и «дневник» для одной и той же поездки.

4. **Сценарий: быстрое «Следующее место»**
   - Открыть экран «Следующее место».
   - Увидеть следующее непосещённое место в текущей поездке.
   - Быстро открыть его на карте / в навигаторе.

5. **Сценарий: Достопримечательности и события**
   - Создать запись о важном событии/достопримечательности (Highlight) «по горячим следам».
   - (Опционально) привязать её к месту и/или поездке.
   - Просмотреть список таких записей за поездку или за всё время.

---

### 2.3. Крупные этапы разработки (MVP)

**Этап 1. Базовая инфраструктура проекта**
- Проверить и при необходимости донастроить Expo‑проект в `mobile/` (TypeScript, Expo Router, React Native Paper).
- Настроить базовую тему, шрифт и общие компоненты (AppBar, кнопки, карточки).
- Спланировать структуру директорий для экранов, компонентов и слоёв данных (модели, сервис работы с БД).

**Этап 2. Локальная база данных и модели**
- Выбрать библиотеку для работы с SQLite в React Native (через Expo).
- Описать модели данных:
  - `Place`;
  - `Trip`;
  - `TripPlace`;
  - `Highlight`.
- Реализовать слой доступа к данным (CRUD‑операции для каждой сущности).
- Настроить миграции/инициализацию таблиц при первом запуске приложения.

**Этап 3. Режим «Места»**
- Создать экран списка мест:
  - отображение основных полей;
  - кнопка добавления нового места.
- Создать экран детали места:
  - просмотр описания, флагов, координат;
  - прикрепление фотографий;
  - кнопка «Открыть на карте/в навигаторе».
- Связать экраны с БД (чтение/запись).

**Этап 4. Режим «Поездки»**
- Создать экран списка поездок.
- Создать экран создания/редактирования поездки (название, даты, описание).
- Реализовать добавление/удаление мест в поездке, изменение порядка маршрута.
- Настроить связь `Trip` ↔ `TripPlace` ↔ `Place` в интерфейсе.

**Этап 5. Режим «Следующее место»**
- Реализовать логику выбора активной поездки.
- Написать функцию определения «следующего места» (первое непосещённое по порядку).
- Создать экран «Следующее место»:
  - краткая информация о месте;
  - кнопки «открыть на карте» и «отметить как посещённое».

**Этап 6. Режим «Достопримечательности»**
- Создать модель и репозиторий для `Highlight`.
- Реализовать экран списка достопримечательностей (фильтрация по поездке/по времени).
- Реализовать экран создания/просмотра записи:
  - свободный текст,
  - привязка к месту/поездке,
  - фото.

**Этап 7. Навигация и общий UX**
- Настроить главный экран с кнопками перехода:
  - Места, Поездки, Достопримечательности, Следующее место, Настройки.
- Обеспечить понятную навигацию между списками и деталями.
- Добавить базовые состояния загрузки/ошибок, пустых списков.

**Этап 8. Полировка MVP**
- Минимальная настройка темной/светлой темы (если уместно).
- Проверка работы офлайн, без доступа к сети.
- Проверка корректности сохранения данных и их восстановления при перезапуске приложения.
