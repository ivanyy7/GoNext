## PROJECT_MAP.md — карта проекта GoNext

Этот файл описывает структуру проекта и то, где что лежит.  
По мере развития проекта карта будет дополняться, но уже созданные описания мы не удаляем.

### 1. Корень репозитория

- `PLAN.md` — исходный общий план и цели проекта.
- `PROJECT_MAP.md` — этот файл, карта структуры.
- `PROJECT_LOG.md` — рабочий журнал шагов и принятых решений.
- `.cursor/` — настройки Cursor и правила взаимодействия с ИИ.
 - `mobile/` — Expo-проект (Expo Router + TypeScript + React Native Paper).

### 2. Крупные директории (будут уточняться по мере работы)

- `mobile/app/_layout.tsx` — корневой лейаут Expo Router, оборачивает приложение в `PaperProvider` и подключает стек навигации; теперь также оборачивает всё приложение в `ThemePreferenceProvider`, чтобы можно было вручную переключать светлую/тёмную тему.
- `mobile/app/index.tsx` — главный экран Home с шапкой `GoNext` и кнопками перехода к основным режимам (Места, Поездки, Следующее место, Достопримечательности, Настройки).
- `mobile/app/settings/index.tsx` — экран «Настройки» (общая информация и опции приложения); содержит карточку «Переключение темы» с иконкой солнца/месяца (по нажатию переключается светлая/тёмная тема) и блок выбора акцентного цвета тёмной темы в виде ряда из 10 цветных кружков.
- `mobile/app.json` — конфигурация Expo-приложения; для сборки через EAS добавлены `android.package` и блок `extra.eas` (projectId).
- `mobile/eas.json` — конфигурация EAS Build: профиль `release` с сборкой Android APK (`buildType: apk`), канал `production`; используется при команде `eas build --platform android --profile release`.
- `mobile/package.json` — зависимости и npm-скрипты мобильного приложения.
 - `mobile/assets/backgrounds/gonext-bg.png` — общее фоновое изображение, используемое на основных экранах приложения.

#### Этап 2 (локальная БД и модели)

- `mobile/models/` — доменные модели данных (`Place`, `Trip`, `TripPlace`, `Highlight`), описанные в виде TypeScript‑типов.
- `mobile/services/database.native.ts` — слой доступа к локальной базе SQLite для мобильных платформ: инициализация таблиц и CRUD‑операции для всех сущностей, реализованные поверх нового async‑API `expo-sqlite` (`openDatabaseAsync`, `getAllAsync`, `runAsync`) с единым вспомогательным `executeSql`.
- `mobile/services/database.web.ts` — упрощённая веб-реализация без SQLite (заглушка для отладки в браузере, данные не сохраняются).
- `mobile/services/photo-storage.native.ts` — сервис для локального хранения фотографий на мобильных платформах: копирует выбранные изображения в папку приложения (`documentDirectory/photos/...`) и возвращает относительные пути для сохранения в БД.
- `mobile/services/photo-storage.ts` — заглушка для web/не‑native сред, чтобы код, вызывающий хранение фотографий, не ломал сборку и запуск в браузере.

#### Этап 3 (режим «Места»)

- `mobile/app/places/index.tsx` — экран списка мест с возможностью перейти к созданию или редактированию места.
- `mobile/app/places/new.tsx` — экран создания нового места (название, описание, флаги, координаты, фотографии).
- `mobile/app/places/[id].tsx` — экран просмотра и редактирования места с кнопкой «Открыть в навигаторе» и возможностью удалить место.

#### Этап 4 (режим «Поездки»)

- `mobile/app/trips/index.tsx` — экран списка поездок с переходом к созданию и редактированию поездок.
- `mobile/app/trips/new.tsx` — экран создания новой поездки (название, даты, описание, признак текущей поездки).
- `mobile/app/trips/[id].tsx` — экран редактирования поездки и маршрута (поля поездки, список мест маршрута с изменением порядка и удалением).
- `mobile/app/trips/[id]/add-place.tsx` — экран добавления существующих мест в маршрут выбранной поездки.

#### Этап 5 (режим «Следующее место»)

- `mobile/app/next-place.tsx` — экран «Следующее место», который определяет активную поездку, выбирает первое непосещённое место маршрута и показывает краткую информацию и действия (открыть в навигаторе, отметить посещённым).

#### Этап 6 (режим «Достопримечательности»)

- `mobile/app/highlights/index.tsx` — экран списка записей-достопримечательностей с фильтром по текущей поездке или по всем записям.
- `mobile/app/highlights/new.tsx` — экран создания новой записи (свободный текст, дата/время, фотографии, привязка к текущей поездке при наличии).
- `mobile/app/highlights/[id].tsx` — экран просмотра отдельной записи с отображением связанной поездки/места и списка фотографий, а также возможностью удалить запись.

#### Общие UI‑компоненты и оформление

- `mobile/components/ui/primary-button.tsx` — основной кнопочный компонент приложения; использует `react-native-paper` в сочетании с `expo-linear-gradient` для мягкой градиентной заливки (в тёмной теме — более тёмные края и светлый центр) и контрастного текста (в тёмной теме текст почти чёрный).
- `mobile/components/ui/screen-background.tsx` — обёртка‑контейнер для экранов: в светлой теме рендерит общее фоновое изображение `gonext-bg.png` через `ImageBackground`, а в тёмной теме отключает картинку и заливает экран однотонным тёмным цветом темы.
- `mobile/components/ui/form-text-input.tsx` — унифицированное текстовое поле форм с логикой «подсказка показывается только пока поле пустое», используемое на экранах Мест, Поездок и Достопримечательностей.
- `mobile/components/ui/labeled-switch.tsx` — переключатель с подписью и подсказкой по тапу на текст, оформленный в молочной подложке; применяется для флагов «Посетить позже», «Понравилось» и признака текущей поездки.
- `mobile/components/ui/milk-card.tsx` — молочная карточка для пустых состояний и важных текстовых блоков; сейчас это почти белая непрозрачная подложка, чтобы текст хорошо читался и в светлой, и в тёмной теме; используется на списках Мест, Поездок, Достопримечательностей, а также на экранах «Следующее место» и «Настройки».

#### Темы и предпочтения пользователя

- `mobile/context/theme-preference.tsx` — контекст `ThemePreferenceProvider` и хук `useThemePreference`, хранящие выбранный пользователем режим (`light`/`dark`) и индекс акцентного цвета тёмной темы (`darkAccentIndex`) поверх системной схемы; используется в `_layout.tsx`, `screen-background.tsx`, `settings/index.tsx` и позволяет переключать тему и акцентный цвет из настроек.

#### Локализация / i18n

- `mobile/i18n/config.ts` — конфигурация интернационализации на базе `i18next` и `react-i18next`: словари для русского и английского языков, инициализация `initI18n()` с чтением сохранённого языка из `AsyncStorage`, функция `changeAppLanguage()` для смены языка и сохранения выбора; используется в `_layout.tsx` и экранах через хук `useTranslation`.

- Актуальный статус: основные повторяющиеся элементы интерфейса (поля ввода, переключатели с подсказками, молочные карточки) вынесены в переиспользуемые UI‑компоненты, поэтому изменения визуального стиля можно вносить централизованно.

- Если структура существенно изменится, старое описание остаётся, а ниже добавляется актуализированный блок с пометкой даты/этапа.
